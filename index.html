This is a test (semi in-production now) about redoing how we normally build and style dropdown lists. Typically we use the <select> element to acheive this, and then use some JS to do something with the selection. Not only is this uber-hard & limiting in terms of styling (this has been another bane of designers/devs), it’s also a farce.

It is afarce because it goes against the inherit nature of the <select> element. The browser expects this to be used inside a <form> element, and it is also not expecting anything to be changed on the page because of a user selection. It is also not expecting to be redirected elsewhere because of said user selection.

Redoing the dropdown

By redoing this as an unordered list with normal hyperlinks inside them, this is way more intuitive to the browser. It is also way easier to style, and can simply made to work cross-browser and on desktop, tablet and mobile. We can easily load in content via AJAX based on the link the user clicks on, or just send them on their merry way. Lastly, if JS is disabled for users, the links and experience will be just as the browsers intended them to be.
